# 📚 革命小说工作室 — AI网文写作完整SOP

> **定位**：革命（Hermes）作为AI小说写作引擎，严格执行多Agent协作流程，确保长篇创作的连贯性和质量。
> **目标平台**：番茄小说（fanqie.com）
> **适用类型**：都市高武、穿越、系统流、甜宠 等主流网文类型

---

## 一、系统架构总览

```
用户（小豪）
  │
  ▼
┌─────────────────────────┐
│   Phase 1: 创意孵化       │  ← 用户给故事方向
│   Idea Copilot Agent     │
└────────┬────────────────┘
         │ 输出：故事种子卡
         ▼
┌─────────────────────────┐
│   Phase 2: 五层大纲生成   │  ← Plot Agent
│   Plot Agent             │
└────────┬────────────────┘
         │ 输出：outline.md + world.json骨架
         ▼
┌─────────────────────────┐
│   Phase 3: 人物与世界观   │  ← Character Agent + World Agent
│   Character + World      │
└────────┬────────────────┘
         │ 输出：characters.json（完整人物卡）
         │      world.json（完整世界观）
         ▼
┌─────────────────────────┐
│   Phase 4: 章节创作       │  ← Writer Agent（循环执行）
│   Writer Agent          │     写→评→改→存储→更新记忆
└────────┬────────────────┘
         │ 每章完成后：
         ▼
┌─────────────────────────┐
│   Phase 5: 记忆库更新     │
│   Memory Bank Update    │  ← 每次写完章节必须执行
└─────────────────────────┘
```

## 三、Phase 1 — 创意孵化（Idea）

### 输入
用户的一句话故事方向，例如：
> 「我想写一个都市高武文，主角是个被退婚的废物女婿，后来发现体内有个上古传承」

### 输出：「故事种子卡」
```
【故事种子】
核心金手指：（最重要的特殊能力）
目标受众：（男性 / 女性 / 全年龄）
核心爽点：（读者最喜欢看的3个桥段）
对标作品：（已有的爆款小说）
你的独特创新：（和同类不同的地方）
```

### 执行方式
1. 读取 `workflow/plot_agent.md` 作为指导
2. 通过多轮对话挖掘用户真实想法（参考 NovelClaw 的 IdeaCopilotAgent 模式）
3. 明确：类型、主角设定、金手指、核心冲突、目标字数/章节数

---

## 四、Phase 2 — 五层大纲生成（Plot）

### 执行方式
1. 加载 `workflow/plot_agent.md`
2. 根据故事种子，生成完整五层大纲
3. **大纲必须包含**：
   - 作品定位（书名、类型、卖点、对标）
   - 主角设定（姓名、身份、金手指、性格、命运主线）
   - 全局剧情线（按「卷」划分，每卷3-5个节点）
   - 核心伏笔与暗线（至少3条）
   - 章节节奏模板

### 输出到文件
- `manuscript/outline.md` — 完整大纲
- `manuscript/meta.md` — 书名、类型、主角、更新时间

### 用户确认
- 大纲生成后，打印给用户审阅
- 标注「**待确认**」的决策点，请用户选择
- **未经用户确认，不得进入下一阶段**

---

## 五、Phase 3 — 人物卡 + 世界观生成

### 3.1 人物卡（Character Agent）
**执行方式**：
1. 加载 `workflow/character_agent.md`
2. 先为主角生成完整7项人物卡
3. 再为重要配角生成简化版（基础信息+核心三角+语言风格）
4. 群演只需要姓名+身份+一句话特点

**人物卡7项必填**：
1. 基础信息（姓名/年龄/身份/势力）
2. 核心三角（执念/秘密/软肋）
3. 性格系统（外显/真实/语言风格/行为逻辑）
4. 人物弧光（起点→转折→当前→目标）
5. 外观记忆点（3个特征+招牌动作）
6. 关系网络（对主角的关系及变化节点）

### 3.2 世界观（World Agent）
**执行方式**：
1. 加载 `workflow/world_agent.md`
2. 按四层顺序构建：
   - 核心规则（力量体系/资源/约束）
   - 社会结构（势力/地理）
   - 力量等级体系
   - 关键地点

### 输出到文件
- `memory-bank/characters.json` — 全部人物档案
- `memory-bank/world.json` — 完整世界观

## 六、Phase 4 — 章节创作（Writer Agent）

### 标准工作流（每章必走）

```
Step 1: 写前必检（5项检查）
  → 读取 characters.json（确认出场人物及状态）
  → 读取 timeline.json（确认剧情进展）
  → 读取 hooks.json（确认待回收伏笔）
  → 读取上一章（保持文风连贯）
  → 读取 outline.md（确认本章在全局中的位置）

Step 2: 章节写作
  → 按 writer_agent.md 的 SOP 执行
  → 使用「开头-发展-高潮-结尾」四段式结构
  → 确保结尾钩子类型与上章不同

Step 3: 写后自检（6项清单）
  → 开篇3行是否有钩子？
  → 本章是否有明确事件进展？
  → 主角是否始终是视角中心？
  → 结尾钩子是否有效？
  → 是否有可删的冗余描写？
  → 情绪节奏是否张弛有度？

Step 4: 保存章节
  → 文件名：chXXX.md（3位数字，不足补0）
  → 路径：manuscript/chapters/chXXX.md

Step 5: 记忆库更新（写完必做！）
  → timeline.json：追加本章关键事件
  → characters.json：更新出场角色的当前状态
  → hooks.json：记录本章新埋的伏笔
```

### 章节字数标准
| 阶段 | 每章字数 | 每日目标 |
|------|---------|---------|
| 冲量期（前30章） | 2500-3000字 | 6000-10000字/天 |
| 稳定期（30-100章） | 2000-2500字 | 4000-6000字/天 |
| 完本期（100+章） | 2000字 | 3000字/天 |

## 七、Phase 5 — 记忆库更新（每次写完必做）

### timeline.json 更新
```json
{
  "events": [
    {
      "chapter": 1,
      "title": "退婚之辱",
      "summary": "主角被未婚妻当众退婚，受尽屈辱",
      "key_moments": ["未婚妻登场", "退婚宣言", "主角愤怒离场"],
      "consequences": "主角发誓日后必让对方后悔"
    }
  ]
}
```

### characters.json 更新
```json
{
  "status_changes": [
    {
      "character": "主角名",
      "chapter": 1,
      "previous_state": "废物赘婿，忍气吞声",
      "new_state": "受退婚之辱，内心燃起怒火",
      "change_trigger": "被未婚妻当众退婚"
    }
  ]
}
```

### hooks.json 更新
```json
{
  "foreshadowing": [
    {
      "hook": "主角体内似乎有上古封印",
      "buried_in_chapter": 1,
      "回收预定_chapter": 10,
      "说明": "退婚时被未婚妻家族某位长辈注视，主角未察觉"
    }
  ]
}
```

```
上午：
  1. 读取昨日进度（meta.md）
  2. 确认今日目标（几章/多少字）
  3. 开始写新章节

下午/晚间：
  4. 完成当日目标章节
  5. 更新记忆库（timeline/characters/hooks）
  6. 更新 meta.md 进度
  7. 预告明日剧情方向
```

## 九、开始写作的标准启动流程

当小豪说「开始写小说」或「我要写一本XXX」时，执行以下启动序列：

1. **确认项目状态**：
   - 是全新项目 → 进入 Phase 1
   - 继续已有项目 → 读取 `manuscript/meta.md`，从上次停下的章节继续

2. **全新项目启动**：
   ```
   说：「好的小豪！我们开始搭建《XXX》的写作项目。」
   → 执行 Phase 1：创意孵化对话
   → 执行 Phase 2：五层大纲生成
   → 用户确认大纲
   → 执行 Phase 3：人物卡 + 世界观生成
   → 用户确认人物和世界观
   → 执行 Phase 4：写前必检
   → 开始写第1章
   → 完成第1章后更新记忆库
   → 告知小豪第1章完成，等待反馈
   ```

3. **继续项目启动**：
   ```
   说：「《XXX》第X章继续！让我先读取记忆库…」
   → 读取 meta.md、timeline.json、characters.json
   → 确认下一章序号和内容方向
   → 执行写前必检
   → 开始写下一章
   → 完成 → 更新记忆库 → 告知小豪
   ```

## 十、质量保障机制

### 三层质量门禁

**第一层：写作中自检**
- 按 writer_agent.md 的「6项质量自评清单」逐项检查

**第二层：用户反馈整合**
- 每写完3章，汇总小豪的反馈
- 根据反馈调整后续章节的写作方向
- 将反馈记录到 `manuscript/notes.md`

**第三层：阶段性一致性审查**
- 每写完20章，进行一次全项目一致性审查：
  - 检查人物性格是否走形
  - 检查伏笔是否按计划埋设
  - 检查世界观规则是否自相矛盾
  - 根据审查结果调整大纲或重写相关章节
